# Pokemon 果实数据表设计（定稿）

> 适用数据源：`docs/fruit/list_*`（字段示例：`id`、`name`、`icon`、`related_pet_id`、`item_quality`）

## 1. 设计目标

- 按最小字段集落库，先满足查询和后续同步需求。
- 以数据源主键 `source_id` 做幂等写入。
- 通过 `pokemon_source_id` 保留与精灵来源ID的关联能力。

## 2. 表结构

### 表名

- `pokemon_fruit`

### 字段定义（PostgreSQL）

```sql
CREATE TABLE IF NOT EXISTS pokemon_fruit (
    id                 SERIAL PRIMARY KEY,                 -- 自增主键
    source_id          BIGINT       NOT NULL,              -- JSON.id
    name               VARCHAR(100) NOT NULL DEFAULT '',   -- JSON.name
    icon               VARCHAR(500) NOT NULL DEFAULT '',   -- JSON.icon
    pokemon_source_id  BIGINT       NULL,                  -- JSON.related_pet_id
    item_quality       SMALLINT     NOT NULL DEFAULT 0,    -- JSON.item_quality
    created_at         TIMESTAMP    NOT NULL DEFAULT NOW(),
    updated_at         TIMESTAMP    NOT NULL DEFAULT NOW(),
    CONSTRAINT uk_pf_source_id UNIQUE (source_id)
);
```

### 索引建议

```sql
CREATE INDEX IF NOT EXISTS idx_pf_pokemon_source_id ON pokemon_fruit (pokemon_source_id);
CREATE INDEX IF NOT EXISTS idx_pf_item_quality ON pokemon_fruit (item_quality);
CREATE INDEX IF NOT EXISTS idx_pf_name ON pokemon_fruit (name);
```

## 3. 字段映射规则

来自 `docs/fruit/list_*` 的单条记录，映射如下：

- `id` -> `source_id`（字符串转整数）
- `name` -> `name`
- `icon` -> `icon`
- `related_pet_id` -> `pokemon_source_id`
- `item_quality` -> `item_quality`
- 其余字段（如 `elem_type_icons`、`evolution_special_condition`）本表不落库

## 4. 约束与数据质量建议

- `source_id` 唯一，作为幂等写入键（`ON CONFLICT (source_id) DO UPDATE`）。
- `item_quality` 建议后续加范围约束（如 `CHECK (item_quality BETWEEN 0 AND 10)`）。
- URL 字段保持原值，不在入库阶段做网络可达性校验（减少导入耗时）。

## 5. 查询场景覆盖

- 按精灵来源ID查看果实：`WHERE pokemon_source_id = ?`
- 按品质筛选：`WHERE item_quality >= ?`
- 按名称搜索：`WHERE name ILIKE '%关键词%'`
- 按外部ID精确查：`WHERE source_id = ?`

## 6. 后续可扩展方向

- 若后续需要与 `pokemon.id` 强关联，可补充 `pokemon_id` 字段并建立外键。
- 若需要审计同步过程，可补 `sync_batch_id`、`last_seen_at` 字段。
- 若前端检索压力增加，可按实际查询补充组合索引（如 `(pokemon_source_id, item_quality)`）。

## 7. 与现有表关系

本设计的 `pokemon_fruit` 为独立“果实”实体表，用于承载 `docs/fruit/list_*` 的果实数据。
