# 精灵性格表（Personality / Nature）

> 数据源：`docs/pets/personalities.json`
> 代码中暂未建表；建议在 PostgreSQL 中新增 `personality` 字典表，并在 `pokemon_instance`（玩家所拥有的精灵实例）上引用 `personality_id`。

## 1. 设计说明

每个精灵在孵化/捕获时会被随机分配一个**性格**，性格会对精灵的六项基础能力产生**乘法修正**：

| 缩写 | 中文 | 字段 |
| --- | --- | --- |
| HP | 体力 | `hp_mod_pct` |
| 物攻 | 物理攻击 | `phy_atk_mod_pct` |
| 魔攻 | 魔法攻击 | `mag_atk_mod_pct` |
| 物防 | 物理防御 | `phy_def_mod_pct` |
| 魔防 | 魔法防御 | `mag_def_mod_pct` |
| 速度 | 速度 | `spd_mod_pct` |

修正规则（与宝可梦一致）：

- 每个性格恰好让**一项能力 +10%**、**另一项 -10%**，其余四项不变。
- 六项两两组合（加/减不同项）共 `6 × 5 = 30` 种，对应现有的 30 条数据。
- 尚未规划"加减同项"（即全中性）性格，如需要可再扩展 6 条（如 Hardy/Docile/Serious/Bashful/Quirky）。

最终战斗属性计算（建议）：

```
final_stat = floor(base_stat * (1 + personality_mod_pct))
```

## 3. 按"加成项"分组速查

### 提升速度（-其它）
- 胆小（-物攻）、开朗（-魔攻）、急躁（-物防）、天真（-魔防）、浮躁（-HP）

### 提升物攻（-其它）
- 固执（-魔攻）、调皮（-魔防）、孤僻（-物防）、坦率（-HP）、勇敢（-速度）

### 提升魔攻（-其它）
- 保守（-物攻）、稳重（-物防）、马虎（-魔防）、认真（-HP）、冷静（-速度）

### 提升物防（-其它）

## 4. 推荐用法（配合 `preferred_attack_style`）

`Pets.json` 里每只精灵有 `preferred_attack_style`（`Physical` / `Magic` / `Both`），可据此推荐性格：

- **物攻系**（Physical）：首选 **固执 / 调皮**，速度不足时考虑 **勇敢**，想抢速时选 **胆小/开朗** 走输出速攻。
- **魔攻系**（Magic）：首选 **保守 / 稳重 / 马虎**，慢速炮台用 **冷静**。
- **双刀（Both）**：推荐减「非主要攻击」，如物理主加魔向则选 **开朗**；视实际技能配比取舍。
- **肉盾 / 奶妈**：HP 或防御向 **大胆 / 慎重 / 沉默 / 平和**。

## 5. 数据库建表建议

```sql
-- 性格字典
CREATE TABLE personality (
    id               SMALLINT PRIMARY KEY,
    name_en          VARCHAR(32) NOT NULL UNIQUE,
    name_zh          VARCHAR(16) NOT NULL UNIQUE,
    hp_mod_pct       NUMERIC(3,2) NOT NULL DEFAULT 0,   -- 例: 0.10 / -0.10 / 0
    phy_atk_mod_pct  NUMERIC(3,2) NOT NULL DEFAULT 0,
    mag_atk_mod_pct  NUMERIC(3,2) NOT NULL DEFAULT 0,
    phy_def_mod_pct  NUMERIC(3,2) NOT NULL DEFAULT 0,
    mag_def_mod_pct  NUMERIC(3,2) NOT NULL DEFAULT 0,
    spd_mod_pct      NUMERIC(3,2) NOT NULL DEFAULT 0
);
```

导入命令可在 `scripts/` 下新增 `import_personalities.py`，从 `docs/pets/personalities.json` 读取后 upsert。
