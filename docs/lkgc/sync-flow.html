精灵图鉴同步功能

基于 lkgc 接口的宠物同步 + 技能同步流程设计

宠物同步 全量/增量
1 请求
fetch_pet_list() → 获取 getList 数据
getPetInfo() → 获取每个宠物的 getPetInfo 详情
2 匹配规则
name getList 的 name
from getList 的 from
组合键 {name}_{from} → 对应 pokemon.name
命中 → 跳过（只插不更新）；未命中 → 新增
3 字段映射 getPetInfo → pokemon 表
name getPetInfo → name image main_url → image（同时上传 OSS） type evolution_stage: 0/3=final, 1=Ⅰ阶, 2=Ⅱ阶 form 默认 original form_name 默认 原始形态 egg_group egg_type_list → 中文蛋组名（见下方对照表） trait_it 特性表新增特性 detail_url main_url image_lc main_url chain_id 默认空 sm / hp hp → 整数，空值=0 wg / atk atk → 整数，空值=0 mg / matk matk → 整数，空值=0 wf / def_val def_val → 整数，空值=0 mf / mdef mdef → 整数，空值=0 sd / spd spd → 整数，空值=0 total_race hp+atk+matk+def_val+mdef+spd 之和 obtain_method position name_en icon_name 去掉 JL_ 前缀 image_yise yise_url avatar 默认空 source_id base_id
4 额外处理
main_url 上传到 OSS
特性在 trait 表新增
egg_type_list → 中文蛋组名 → 写入 pokemon.egg_group + pokemon_egg_group
技能同步 → 见右侧「技能同步」
技能同步 弹窗操作
1 触发
在精灵维护列表的操作列点击「同步技能」按钮
2 匹配当前宠物
条件 pokemon.name = getList 的 {name}_{from}
获取 getPetInfo → skill_list
3 技能分类
base → type=原生技能 bloodline → type=血脉技能 stone → type=技能石技能
4 技能匹配（skill 表）
命中 复用 skill.id，不更新 skill 属性
未命中 新增 skill 记录
新增 skill 字段映射
take_energy consume power power desc skill_desc icon_url 上传 OSS 后写入 attr_id 先写 NULL（以本库维护为准）
技能类型推断（无本库值时）
category_id=1 + damage_id=1 物攻 category_id=1 + damage_id=2 魔攻 category_id=2 + damage_id=-1 防御 category_id=3 + damage_id=-1 状态
tag_id 不作为类型映射依据
5 关系写入 pokemon_skill 表
幂等键 (pokemon_id, skill_id)
重复时 更新 type 和 sort_order
6 结果弹窗
请求技能数接口返回的总数 命中/新增技能数skill 表匹配/创建数 新增/更新关系数pokemon_skill 变更数 跳过数无需处理的条数 告警信息异常或无配置提示 逐条结果每个技能的处理详情

注意：一只宠物的 egg_type_list 可能有多个值（数组），对应多蛋组

字典表：lkgc_egg_group（id + name），类似 lkgc_skill_type 的用法

存储策略
egg_type_list → 2 个写入目标：
pokemon.egg_group (VARCHAR)
逗号拼接所有蛋组名，如 动物组,拟人组
pokemon_egg_group (多对多表)
每个蛋组名一条记录：(pokemon_id, group_name)
关键规则汇总
只新增，不更新已有 pokemon 数据
同步技能时复用已有 skill，不覆盖属性
技能属性以本库维护为准（先写 NULL）
特性同步时在特性表新增
main_url / icon 上传 OSS
